animByNameExtra()method
public static function animByNameExtra(target:ixDef, animName:String, onComplete:Object = null, killCurrent:Boolean = true, reset:Boolean = false, doNotDisturb:Boolean = false):int

Executes animation on meta[animName] owners. Lenient. General format of an animation is an array of arguments for axl.utils.AO.animate function with difference that first parameter (target) is ommited, as it is passed to this function directly. Following example is the shortest example of animating object from whichever position to x 10 in 0.3sec every time object is added to stage. 

		 <div name='a' meta='{addedToStage:[0.3,{"x":10}]}'/>
Mapped properties of inner animation object: onUpdate, onUpdateArgs

Parameters

target:ixDef — - object to animate and look meta[animName] for
 
animName:String — - animation key/identifier
 
onComplete:Object (default = null) — - callback to execute once animation is completd
 
killCurrent:Boolean (default = true) — - kills ANY existing animations proceeding on target befeore executing this one
 
reset:Boolean (default = false) — can execute ixDef reset interface function before animation
 
doNotDisturb:Boolean (default = false)

applyAttributes()method 
public static function applyAttributes(def:XML, target:Object):Object

Maps XML attributes to target properties. If target (any object) does not own property specified as attribute in XML definition, then property is not assigned. If it does, value of attribute is arsedand assigned to the target's property. If object is dynamic and property of given attribute name does not exist, new property on target is not created. Parsing attribute values is proceed by resolveValue function The order of the attributes matters - they're being processed from left to right. Attributes can reffer to object's deeper properties.

drawFromDef()method 
public static function drawFromDef(def:XML, drawable:Sprite = null):DisplayObject

XML tag name graphics expects command children only. Draws on canvas of drawable DisplayObject (if flash.display.graphics Class is in its scope). Each command node represents one instruction to flash.display.Graphics class instance where node's value is the name of the class function and node's attrubutes are arguments for that function Eg. 

		 <graphics>
		 	<command color='0x9bde43' alpha='0.7'>beginFill</command>
		 	<command x='0' y='0' width='148' height='44'>drawRect</command>
		 </graphics>
is equivalent of 
		 drawable.graphics.beginFill(0x9bde43,0.7);
		 drawable.graphics.drawRect(0,0,148,44);
Attribute names are not important, unlike it's order.

getReadyType2()method 
public function getReadyType2(xml:XML, callBack:Function = null, decorator:Function = null, callBack2argument:Object = null, xroot:xRoot = null):void

Translates xml node to an ActionScript object within predefined types and their equivalents:

  • img - axl.xdef.types.display.xBitmap extends flash Bitmap
  • div - axl.xdef.types.display.xSprite - extends flash Sprite
  • txt - axl.xdef.types.display.xText - extends flash TextField
  • btn - axl.xdef.types.display.xButton- extends xSprite
  • act - axl.xdef.types.xAction- function equivalent - lightweight code container
  • msk - axl.xdef.types.display.xMasked - extends xSprite
  • swf - axl.xdef.types.display.xSwf - loaded flash DisplayObject is added to xSwf as a child
  • data - axl.xdef.types.xObject - loaded data is being analyzed and can be instantiated as XML ('xml'), Object ('json'), Sound ('mp3','mpeg'), DisplayObject ('jpg','png','gif','swf') or raw data (ByteArray, String). Regardles of it's contents, instantiated axl.xdef.types.xObject assigns the result to it's own data property.
  • scrollBar - axl.xdef.types.display.xScroll - extends xSprite
  • carousel - axl.ui.Carusele extends flash Sprite
  • script - axl.xdef.xScript extends xObject - allows to load and merge external config files

Parameters

xml:XML — - XML object which tag name matches one of the listed elements. Attribute src will delay calling callback to the moment resource is available. Resources are being loaded by axl.utils.Ldr.load() method. If resource is already loaded, and available within Ldr.getAny - no reloading is performed and callback called quicker. All Bitmap related objects requested within this method are being re-drawn from originaly loaded resource (copy). To avoid duplicating objects use getAdditionByName function, which controlls cache of loaded elements and groups.
 
callBack:Function (default = null) — - function to execute once element is available. It should accept one parameter - loaded element, or two arguments if callBack2argument is specified.
 
decorator:Function (default = null) — - Function that each and every instantiated object (all descendands) will be passed to as an argument
 
callBack2argument:Object (default = null) — - optional second argument for callback. It is in use to pushReadyTypes children order.
 
xroot:xRoot (default = null) — - xRoot reference that result of this function will belong to

resolveValue()method 
public static function resolveValue(val:String, target:Object = null):*

Generally all XML attribute values are recoginzed as String values (ActionScript), and this is an ultimate value returned IF parsing fails. There are two ways of parsing val available:

  • dolar sign prefixed parsing - value is evaluated by axl.utils.RootFinder.parseInput method
  • JSON parsing - value assigned to a target is an output of JSON.parse on attribute value 
		 <div name='sample' x='10' y='$stage.stageHeight/2' mouseChildren='false'/>
is equivalent of 
		 var any:xSprite = new xSprite()
		 any.name = 'sample';
		 any.x = 10;
		 any.u = stage.stageHeight/2;
		 any.mouseChildren = false;
