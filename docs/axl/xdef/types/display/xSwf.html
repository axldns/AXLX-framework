Classpublic class xSwf
InheritancexSwf xSprite flash.display.Sprite

Class is container for loaded swf files providing API for controling timeline animations. Instantiated from: General convention that loaded swf has to follow in order for playback API to work is to keep entire animation in MainTimeline rather than inside inner MovieClip objects. Different approach may involve accessing MovieClip itself and calling it's inner API code-wise.

stopOnEndproperty 
public var stopOnEnd:Boolean = false

Determines if stop() should be called on swf when it reaches end value. End value varies and depends on different settings:

  • played normally reaches max number of frames or reaches frame stopOnFrame
  • played on reverse and reaches frame 1 or reaches frame stopOnFrame
If set to false but stopOnFrame is greater than zero, stop will also be called with under tha same conditons as above.

The default value is false.

See also

stopOnFrame
stopOnEndAfterXcyclesproperty 
public var stopOnEndAfterXcycles:int

Determines how many time swf should reach it's end value before it stops. End value can be max number of frames, or frame 1 (yoyo, reverse). To make this property working stopOnEnd must be set to false.

See also

stopOnEnd
stopOnFrameproperty 
public var stopOnFrame:int = -1

In typical scenario determines on which frame swf should be stopped, but may depend on other settings: stopOnEnd=true - stops playback promptly exactly on the frame number assigned to this property, stopOnEnd=false - depends on stopOnEndAfterXcycles. stopOnEndAfterXcycles>0 - stopOnFrame determines "last frame of single cycle", after x cycles MovieClip is stopped on either frame 1 or total number of frames. stopOnEndAfterXcycles is not set - loops. This property does not apply if yoyo=true #default-1
