Packageaxl.xdef.types.display
Classpublic class xScroll
InheritancexScroll xSprite flash.display.Sprite

Class provides simple scroll bar functionality. Instantiated from <scrollBar/>. It makes use of axl.ui.controllers.BoundBox controler, which needs two Display Objects to satisfy functionality (bound & box), here named as rail and train. Scroll bar instance recognizes Display Obecjts added to its display list, and auto-links functionality if their name matches pattern. List of (case insensitive) names recognized:
  • rail
  • train
  • btnLeft
  • btnRight
  • btnDown
  • btnUp
Since need of access the buttons through the registry and having multiple scroll bar instances at the same time, some of "rail", "train" and other may simply become unavailable. Thus official assignments should be made, eg.:scrollBar.rail = this;scrollBar.train = that; Scroll bar can have only one orientation at the time, hence btnIncrease and btnDecrease should be subject of assignments onward while setting controller's orientation to both horizontal and vertical may result in unexpected output. Controller's initial values can be overriden, default values as follows: ctrl.bound = rail;ctrl.box = train;ctrl.horizontalBehavior = BoundBox.inscribed; ctrl.verticalBehavior = BoundBox.inscribed;ctrl.liveChanges = true;
