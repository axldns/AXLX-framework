Packageaxl.xdef.types.display
Classpublic class xButton
InheritancexButton xSprite flash.display.Sprite

Typical interactive button class. Does not require adding any event listeners - all listeners are added internally - requires to define callback functions instead. Allows to set of custom actions and states and common styling by providing just a textual reference. Actions can be defined on click, on hover, on hover out, on press, on release. States: over, out, enabled, disabled. Hover and press functions execution can be repeated in certain intervals just by defining number of miliseconds gap between execution.

statesproperty 
states:String

Allows to set displayable objects as button states (over, out, down, disabled). Value for this property should match name of the element available within config's additions node. Element should contain one or more children with name(s) matching state's name(s) (out,over,down,disabled). Children of it are instantiated and pushed to button's display list, where internal filter assigns these to particular button states. Assigning the same style name more than once does not cause multiple instantiation. Asigning diferent button style during runtime removes old states from buttons display list but does not dispose them. Only one state is displayed at the time. It is available to define some states while not defining others without quirky results. Asigning same style to different buttons does not cause "stealing" children.Example style: <div name='standardButton'> <img name='out' src='/assets/btnOut.png'> <img name='over' src='/assets/btnOver.png'> </div>

execute()method 
public function execute(e:MouseEvent = null):void

Performs main execution (code attribute + meta keywords) in pre-defined order. Typically called automatically on event of CLICK (not mouse down), but this can be dependent on few factors:

  • If externalExecution is defined - execute won't be called on CLICK
  • If executeReleasedOutside = true - execution will be called even though CLICK event isn't fired in this case.
For example xForm instances by default automatically detect xButton instances and "intercept" their functionalty by defining externalExecution where internal form validation is checked and execute is called only when passed.
  1. meta.url - if defined, navigates to url. Can be URL string or an Array where first element is URL String, second "window type",eg. "_self"
  2. meta.js - if defined and available, performs ExternalInterface.call, where js should supply an Array of arguments for call function
  3. meta.post - if defined, used as an Array of arguments for sendData function from ConnectPHP class
  4. code - evaluates portion of uncompiled code assigned to it, String argument for binCommand function
